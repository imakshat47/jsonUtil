🎨 Generate HTML Forms

Basic Form (No Framework)

Input JSON:

{
  "name": "John Doe",
  "email": "john@example.com",
  "age": 30,
  "isActive": true
}

Request:

curl -X POST http://localhost:4000/v1/forms \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "X-Operation: FORM" \
  -H "X-Response-Type: html" \
  -H "X-CSS-Framework: none" \
  -H 'X-JSON-Payload: {"name":"John Doe","email":"john@example.com","age":30,"isActive":true}'

Generated HTML:

<div class="form-group">
  <label for="name" class="form-label">Name</label>
  <input type="text" class="input-field" id="name" name="name" value="John Doe">
</div>
<div class="form-group">
  <label for="email" class="form-label">Email</label>
  <input type="text" class="input-field" id="email" name="email" value="john@example.com">
</div>
<div class="form-group">
  <label for="age" class="form-label">Age</label>
  <input type="number" class="input-field" id="age" name="age" value="30">
</div>
<div class="form-group">
  <label for="isActive" class="form-label">IsActive</label>
  <input type="checkbox" class="input-field" id="isActive" name="isActive" checked>
</div>

📋 Generate JSON Schemas

Schema Generation

Input JSON:

{
  "user": {
    "profile": {
      "name": "Alice Johnson",
      "age": 28
    },
    "preferences": {
      "newsletter": true,
      "theme": "dark"
    }
  },
  "timestamp": "2025-01-15T10:30:00Z"
}

Request:

curl -X POST http://localhost:4000/v1/forms \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "X-Operation: FORM" \
  -H "X-Response-Type: json" \
  -H 'X-JSON-Payload: {"user":{"profile":{"name":"Alice Johnson","age":28},"preferences":{"newsletter":true,"theme":"dark"}},"timestamp":"2025-01-15T10:30:00Z"}'

Generated JSON Schema:

{
  "type": "object",
  "properties": {
    "user": {
      "type": "object",
      "properties": {
        "profile": {
          "type": "object",
          "properties": {
            "name": {
              "type": "string"
            },
            "age": {
              "type": "number"
            }
          }
        },
        "preferences": {
          "type": "object",
          "properties": {
            "newsletter": {
              "type": "boolean"
            },
            "theme": {
              "type": "string"
            }
          }
        }
      }
    },
    "timestamp": {
      "type": "string"
    }
  }
}

✅ Validate JSON

Successful Validation

Request:

curl -X POST http://localhost:4000/v1/forms \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "X-Operation: VALIDATE" \
  -H 'X-JSON-Payload: {"name":"Bob Wilson","age":35,"email":"bob@example.com"}'

Response:

{
  "valid": true,
  "errors": []
}

Validation with Errors

Note: This example shows conceptual validation errors. Actual validation depends on schema inference.

Response (example):

{
  "valid": false,
  "errors": [
    {
      "instancePath": "/age",
      "schemaPath": "#/properties/age/type",
      "keyword": "type",
      "params": {"type": "number"},
      "message": "must be number"
    }
  ]
}

📊 Access Logs

Retrieve Request Logs

Request:

curl -X GET http://localhost:4000/v1/logs \
  -H "Authorization: Bearer YOUR_API_KEY"

Response:

[
  "{\"time\":\"2025-01-15T10:30:00.123Z\",\"method\":\"POST\",\"url\": \"/v1/forms\",\"headers\": {\"authorization\":\"Bearer ***\",\"x-operation\":\"FORM\"}}",
  "{\"time\":\"2025-01-15T10:31:15.456Z\",\"method\":\"POST\",\"url\": \"/v1/forms\",\"headers\": {\"authorization\":\"Bearer ***\",\"x-operation\":\"VALIDATE\"}}"
]

Frontend Processing:

// JavaScript example
const response = await fetch('/v1/logs', {
  headers: { 'Authorization': 'Bearer YOUR_API_KEY' }
});
const logs = await response.json();

logs.forEach(logString => {
  const logEntry = JSON.parse(logString);
  console.log(\`\${logEntry.time}: \${logEntry.method} \${logEntry.url}\`);
});

💡 Tips & Best Practices

🔑 Environment Setup

  • Use strong, unique values for API_KEY and SYSTEM_KEY
  • Keep SYSTEM_KEY stable to preserve log decryption
  • Use different keys for different environments

🎯 Form Generation

  • Use meaningful field names in your JSON
  • Choose the CSS framework that matches your frontend
  • Test with nested objects for complex forms

🔍 Debugging

  • Use the logs endpoint to debug request issues
  • Check validation errors for schema mismatches
  • Test with small JSON payloads first

🚀 Production

  • Implement log rotation for large applications
  • Use HTTPS in production environments
  • Consider rate limiting for public APIs
