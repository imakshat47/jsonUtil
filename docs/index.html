Convert Any JSON to Forms in Real-Time

A backend utility that accepts arbitrary JSON payloads at runtime and generates HTML forms or JSON Schemas with validation, stateless CRUD operations, and encrypted request logging.

üöÄ Quick Start

git clone https://github.com/youruser/jsonutils
cd jsonutils
npm install
cp .env.example .env  # Set API_KEY and SYSTEM_KEY
npm start  # ‚Üí http://localhost:4000

üí´ How It Works

1

Send JSON Payload

POST to /v1/forms with your JSON in X-JSON-Payload header

2

Choose Output

Set X-Operation to FORM, VALIDATE, or CRUD operation

3

Get Results

Receive HTML form, JSON Schema, validation results, or CRUD echo

üìã Example Request

Input JSON:

{"name": "Alice", "age": 25, "email": "alice@example.com"}

cURL Command:

curl -X POST http://localhost:4000/v1/forms \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "X-Operation: FORM" \
  -H "X-Response-Type: html" \
  -H "X-CSS-Framework: bootstrap" \
  -H 'X-JSON-Payload: {"name":"Alice","age":25,"email":"alice@example.com"}'

Generated HTML Form (Bootstrap):

<div class="mb-3">
  <label for="name" class="form-label">Name</label>
  <input type="text" class="form-control" id="name" name="name" value="Alice">
</div>
<div class="mb-3">
  <label for="age" class="form-label">Age</label>
  <input type="number" class="form-control" id="age" name="age" value="25">
</div>
<div class="mb-3">
  <label for="email" class="form-label">Email</label>
  <input type="text" class="form-control" id="email" name="email" value="alice@example.com">
</div>

üèÅ Getting Started

Requirements

  • Node.js 18+ and npm
  • Git for cloning the repository

Installation Steps

  1. Clone the repository: 
    git clone https://github.com/youruser/jsonutils
cd jsonutils
  2. Install dependencies: 
    npm install
  3. Configure environment: 
    cp .env.example .env

    Edit .env and set:

    • PORT=4000 (or your preferred port)
    • API_KEY=your-secret-api-key
    • SYSTEM_KEY=your-encryption-key
  4. Start the server: 
    npm start
  5. Test the API:

    Visit http://localhost:4000/v1 to see the status endpoint.
